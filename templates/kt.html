1. Architecture Overview

ARISE is a state-driven EASM toolkit that converts a list of target domains into a complete exposure map. The system has three layers:

hosts.txt
Target domains
→
arise.py
Orchestrator
→
easm-pipeline.sh
18-Module Pipeline
→
manifest.json
State Store
→
dashboard.py
Flask Dashboard

Data Flow

  1. arise.py reads hosts.txt, creates a timestamped scan directory under scans/, launches the pipeline
  2. easm-pipeline.sh runs 18 modules sequentially, writing results to numbered subdirectories and updating manifest.json
  3. dashboard.py reads scan directories, parses all data sources (manifest, nuclei JSON, nmap services, httpx tech, cloud findings), and serves a real-time dashboard on port 5000
  4. Each scan is isolated: scans/{domain}-{DDMMYY}-{HHMM}/ with its own manifest, logs, and module output folders

4. Risk Scoring Model

ARISE uses a 6-domain composite risk model. Each host gets scored across these domains, clamped to 0–100:

DomainWeightSignals
Vulnerability35Nuclei critical/high findings, confirmed SQLi/XSS/LFI
Credential / Secret25Exposed API keys, tokens, passwords, service account keys
Infrastructure Posture20Missing security headers, open sensitive ports, no WAF
SSL / Domain10Dangling CNAMEs, expired certs, missing HSTS
Attack Surface Breadth10Total exposed endpoints, subdomains, services
Cloud Exposure40Public buckets, exposed metadata, Terraform state, open K8s APIs

Risk Bands

CriticalScore ≥ 70Immediate remediation required
HighScore 50–69Prioritize for next sprint
MediumScore 30–49Track and plan remediation
LowScore < 30Monitor during regular assessments

Port Risk Tiers

Tier 1 (Critical)Redis (6379), Elasticsearch (9200/9300), MongoDB (27017), CouchDB (5984), Memcached (11211)
Tier 2 (High)MySQL (3306), PostgreSQL (5432), MSSQL (1433)
Tier 3 (Medium)RDP (3389), SSH (22), Telnet (23), SMB (445), FTP (21), SMTP (25), POP (110), IMAP (143)

6. API Security Testing

Autoswagger

  • Discovers OpenAPI/Swagger specs automatically from known paths
  • Invocation: autoswagger <spec-url> -json -risk
  • -json is a boolean flag (output to stdout), -risk enables write-method testing

RESTler (Microsoft)

  • Stateful REST API fuzzing: compile → test → fuzz-lean → fuzz
  • --workingDirPath is a global flag placed before the subcommand
  • RESTler is a robustness oracle — it detects HTTP 500s and internal errors, NOT auth bypasses
  • Parses results from runSummary.json and errorBuckets.json (JSON primary, text fallback)
  • Supports auth via APISEC_AUTH_TOKEN_CMD and APISEC_AUTH_REFRESH_SEC env vars

7. Dashboard & API Endpoints

The Flask dashboard serves at http://localhost:5000 with these views:

8. Operations & CLI

# Full scan + dashboard
python3 arise.py

# Dashboard only (browse previous scans)
python3 arise.py --dashboard-only

# Scan only (no web UI)
python3 arise.py --no-dashboard

# Custom targets
python3 arise.py --hosts targets.txt

# Skip modules
python3 arise.py --skip waf_detection --skip param_fuzzing

# Dry run (preview execution order)
python3 arise.py --dry-run

# Custom dashboard port
python3 arise.py --dashboard-port 8080

# Tuning via env vars
python3 arise.py --env PORT_SCAN_MODE=fast --env NAABU_RATE=5000
